Deep Dive: Why Separate Files?

The Analogy: The Recipe Book

Imagine cooking a complex meal. You could write everything on one napkin: ingredients, cooking steps, presentation tips.

But it's much cleaner to have three separate sections: Ingredients (HTML structure), Cooking Steps (JavaScript logic), and Plating Guide (CSS styling).

When you need to change the presentation, you only touch the plating guide - not the whole recipe!

The Technical Reality

Separation of Concerns is a fundamental principle in software development. Each file has one clear responsibility:

HTML (Structure): The skeleton. Defines what elements exist (buttons, cards, inputs).
CSS (Styling): The skin. Defines how elements look (colors, spacing, animations).
JavaScript (Behavior): The muscles. Defines what happens when users interact (clicks, form submissions).
üí° Why it matters for SEs: When demoing to customers, you'll need to update UI colors or add features quickly. With separate files, you can confidently edit styles.css without breaking the logic in app.js.

AI Tools (Your Co-Pilots)

Google Gemini (The Generator)

What: A conversational AI that generates complete code from natural language prompts.
Why: You describe what you want ("create a checklist with 5 sections and a summary button"), and Gemini writes the HTML, CSS, and JavaScript for you. Perfect for rapid prototyping.

GitHub Copilot (The Refiner)

What: An AI assistant built into VS Code that suggests code completions and improvements.
Why: While you're editing, Copilot can add inline comments, suggest better variable names, or autocomplete repetitive code patterns. It helps make your code production-ready.

Live Preview (The Tester)

What: A VS Code extension that shows your HTML page in a live browser window.
Why: Every time you save a file, the preview updates instantly. No need to manually refresh your browser - see changes in real-time as you code.

Project Structure (Lab 2 & 3)

  • index.html The main structure. Contains the checklist sections, buttons, and content containers. Links to CSS and JS files.
  • styles.css All the visual styling. Colors, fonts, spacing, animations. Change this file to rebrand the tool for different customers.
  • app.js Interactive logic. Handles checkbox clicks, generates summaries, copies text to clipboard. The "brain" of the tool.
  • Linking Files In index.html, add:
    <link rel="stylesheet" href="styles.css">
    <script src="app.js"></script>
  • localStorage Browser's built-in storage. Save checkbox states so they persist when you close/reopen the page. No backend needed!

Coding Concepts for SEs

DOM (Document Object Model)

Think of this as the Live Map of your HTML page. JavaScript uses document.getElementById() to find and manipulate elements (like checking a checkbox or updating text).

Event Listeners

The Watchers that trigger code when users interact. addEventListener('click', handleClick) runs the handleClick function whenever someone clicks a button.

localStorage

The browser's Memory Box. Save data that persists even when the page is closed: localStorage.setItem('checked', 'true'). Perfect for saving user preferences!
